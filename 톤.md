# 톤

> **담당자**: @Finn
>
> 톤(Tone)은 상황에 따라 달라지는 말투, 표정과 같습니다. 일반적인 상황에서는 기본 톤을 사용하지만, 특수한 상황에서는 변화된 톤을 사용할 수 있습니다.

## 1. 일반적인 상황

| 원칙 | 설명 |
| --- | --- |
| 해요체 사용 | '해요체'를 기본으로 사용합니다. (~해요, ~이에요, ~예요) |
| 긍정적 표현 사용 | '즉시, 필수, 반드시, 오늘만' 같은 압박·강요 표현은 사용하지 않습니다. |
| 사용자 중심 표현 사용 | 사용자 관점으로 씁니다. 예) '받아가세요' ⭕ '드립니다' ❌ / '구독이 시작됐어요' ⭕ '구독 신청이 완료됐어요' ❌ |
| 쉬운 표현 사용 | 한자어, 내부 용어, 시스템 언어를 피하고 누구나 이해할 수 있는 표현을 사용합니다. |
| 명령어 사용 금지 | '~하세요' 대신 '~해 주세요'를 사용합니다. |
| 주체 높임말 지양 | '~하시겠어요?'처럼 주체·객체 높임말은 사용하지 않습니다. |
| 수동태 사용 금지 | '읽혀지지 않는' → '읽히지 않는' 능동 표현을 사용합니다. |
| 과도한 수식어 금지 | '밀리에만 있는 밀리 오리지널 콘텐츠' ❌ / '밀리에만 있는 콘텐츠' ⭕ |
| 불필요한 단어 제거 | 단어 반복, 중복 의미, 빈 화면을 채우기 위한 말을 쓰지 않습니다. |

## 2. 특수한 상황

### 2-1. 문제/오류 발생 상황

| 구분 | 상황 예시 | 톤 원칙 |
| --- | --- | --- |
| 불편 발생 | 경품 지연, 정책 변경, 서비스 오류 (서비스 이용에 불편함이 생겼을 때) | 사과보다 양해 요청 우선 |
| 사고 발생 | 서비스 장애, 개인정보 유출 (피해 발생) | 명시적 사과 필수 |

**공통 원칙**

- 공감 → 책임 → 다음 행동 순서로 구성
- 다음 행동(해결 방향)은 반드시 포함
- 우선순위: 해결 중심 → 양해 표현 → 사과

### 2-2. 제안/권유 상황

**해당 상황**: 가입, 구독, 콘텐츠 추천, 이벤트, 해지 취소, 개인정보 수집, 수신 동의

**톤 원칙**

- 부드럽고 친근하게, 단 강요 없이
- 선택지를 열어두는 표현 사용
- 공급자에게 불리한 선택지도 숨기지 않는다 (솔직함)
- "지금 아니면 안 된다" 식의 압박 표현 금지

### 2-3. 중요 정보 안내 상황

**해당 상황**: 공지, 유의사항, 대외기간 안내, 약관/정책, 사용자 조사(오픈서베이 등)

**톤 원칙**

- 차분하고 명확하게
- 사용자가 알아야 할 정보만 객관적으로 전달
- 제안·요구 없이 정보 전달에만 집중
- 내용이 길어도 괜찮으나, 문장 단위는 짧게 유지

### 2-4. 설명/튜토리얼 상황

**해당 상황**: 튜토리얼, 신규 기능 소개

**톤 원칙**

- 간결하게 — 설명이 복잡하면 이탈 위험 증가
- 공급자 내부 용어는 반드시 풀어서 표현
- 단순 기능 설명이 아닌 존재 이유 + 사용자 이점 중심으로 설명
