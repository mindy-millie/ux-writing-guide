# 🐶 말티쥬 교정 사례집

말티쥬가 학습하는 실제 교정 사례 모음입니다.
**사례가 쌓일수록 말티쥬의 답변이 정확해집니다.** 좋은 교정 사례가 생길 때마다 아래 형식으로 추가해 주세요.

작성 규칙:

- 한 사례는 `## 사례` 제목 하나로 시작합니다.
- **상황**(어디에 쓰는 문구인지), **원문**, **수정**, **포인트**(왜 이렇게 바꿨는지) 4가지를 적습니다.
- 포인트는 말티쥬가 따라 할 판단 기준이므로 구체적으로 적을수록 좋습니다.

---

## 사례 1. 네트워크 에러

- **상황**: 네트워크 에러가 발생했을때 표시되는 alert
- **원문**: 타이틀 : 네트워크 에러 / 본문 : 네트워크가 오프라인 상태입니다. / 버튼 : 확인
- **수정**: 타이틀 : 인터넷 연결 오류 / 연결 상태를 확인한 뒤 다시 시도해 주세요. / 버튼 : 확인
- **포인트**:
  - 헤더/다이얼로그 타이틀은 짧은 명사형 사용
  - 시스템 중심 표현 지양 (오프라인 상태입니다)
  - 사용자가 지금 무슨 행동을 해야하는지가 없음

## 사례 2. 도서 다운로드 수량 제한 안내 

- **상황**: 전체 다운로드 클릭 시 한번에 다운로드 할 수 있는 도서의 최대 개수가 30개임을 알려주는 alert
- **원문**: 타이틀 : 다운로드 안내 / 본문 : 최대 30개까지 한 번에 다운로드 가능합니다. 초과 수량을 제외하고 다운로드 합니다. / 버튼 : 취소,확인
- **수정**: 타이틀 : 다운로드 기준 안내 / 본문 : 한 번에 최대 30개까지 다운로드할 수 있어요. 전체 항목 중 30개만 다운로드해요.  / 버튼 : 취소,다운로드하기
- **포인트**:
  - 시스템 설명보다 사용자 결과 중심
  - '초과 수량을 제외하고' 라는 내용은 한자어가 너무 많이들어가서 어렵게 느껴지고, 내부 처리 표현이라 제거
  - 현재 상황에 대해서 쉽게 설명
  - 버튼은 '확인' 대신 실제 액션명 사용


## 사례 3. 해지 신청을 중도에 취소

- **상황**: 해지를 신청한 상황에서 중도에 해지 신청을 취소하는 상황에서 표시되는 alert
- **원문**: 타이틀 : 해지 취소 / 본문 : 해지를 취소하고 정기구독을 다시 시작해보세요. 기존 자동결제일에 결제가 되며 구독기간은 동일하게 적용됩니다. 다음 결제일 : 2026.05.22 / 버튼 : 취소,구독시작
- **수정**: 타이틀 : 해지를 취소할까요? / 본문 : 다음 결제일에 결제가 진행되며 구독 요금과 기간은 유지돼요. 다음 결제일 : 2026.05.22 / 버튼 : 다음에 하기,해지 취소하기
- **포인트**:
  - 사용자 입장에서 이해하기 쉽게 설명 : 해지 신청이 취소된다. 다음 결제일에 그대로 결제된다
  - 시스템 설명 느낌 제거


  ## 사례 4. 통합 ID 변경 필요 안내 modal

- **상황**: 비밀번호 확인 절차에서 통합 아이디로 변경을 요청하는 modal
- **원문**: 타이틀 : 계정 정보 확인 필요 / 본문 : 개인정보 수정 시 새로운 통합 아이디 설정이 필요합니다. 원활한 서비스 이용을 위해 인증 후 아이디와 비밀번호를 설정해주세요. / 버튼 : 취소, 확인
- **수정**: 타이틀 : 영문 아이디 설정 필요 / 본문 : 개인정보를 수정하려면 영문 아이디와 비밀번호가 필요해요. 다음 단계에서 본인 인증 후 설정해 주세요. / 버튼 : 닫기, 본인 인증하기
  - 현재 문구는 '통합 아이디' 개념이 갑자기 나와서 이해가 어려움
  - “원활한 서비스 이용”은 추상적이고, 왜 해야 하는지보다 절차 설명 느낌이 강함
  - 버튼 '확인'도 행동 의미 부족
  - 사용자 입장에서 필요한 안내로 변경 '본인 인증 후 아이디/비밀번호 만든다'
  - 어떤 화면으로 이동하는지 예측이 가능하도록 버튼명을 '본인 인증하기' 로 변경


## 사례 5. 난수번호로 된 구독권 등록 완료 modal

- **상황**: 난수번호로 된 이벤트성 구독권을 등록하고 난 이후 표시되는 modal 등록 후 사용자가 구독권 사용하기 버튼을 클릭해야 구독권한이 실행되는 상황
- **원문**: 타이틀 : 구독권 등록 완료 / 본문 : 구독권이 정상 등록되었습니다. 등록된 구독권 목록에서 구독권 사용 처리 후 밀리의서재를 이용하세요 / 버튼 : 확인
- **수정**: 타이틀 : 구독권 등록 완료 / 본문 : 유효 기간이 끝나기 전에 사용하기를 눌러 구독권을 사용해보세요. / 버튼 : 확인
- **포인트**:
  - 사용자 입장에서 핵심 : 등록은 완료됨, 아직 구독 시작은 아님, 사용하기 버튼을 눌러야함, 구독권에는 유효기간이 있음
  - 시스템 메시지 느낌 제거 : '정상 등록되었습니다.', '사용 처리 후'

## 사례. 스토리로드 작가 회원탈퇴

- **상황**: 스토리로드 독점 작품 보유한 경우 회원 탈퇴할때 안내 메시지
- **원문**: 타이틀 : 회원 탈퇴 안내 
내용 : 스토리로드에 독점 제공 작품이 있습니다. 탈퇴를 진행하는 경우 작품은 삭제되지 않습니다. 탈퇴를 진행하시겠습니까? 
버튼 : 취소,확인
- **수정**: 타이틀 : 회원 탈퇴 안내
내용 : 스토리로드에 독점 제공 중인 작품이 있어요. 회원 탈퇴 후에도 작품은 삭제되지 않고 계속 제공됩니다. 회원 탈퇴를 진행할까요?
버튼 : 취소, 탈퇴하기
- **포인트**:
  - 현재 문구는 의미는 전달되지만 조금 딱딱하고, "작품은 삭제되지 않습니다."가 사용자가 왜 알아야 하는지 바로 와닿지 않을 수 있음
  - UX 라이팅 원칙에 맞게 상황 → 영향 → 행동 순서로 정리
  - '확인' 버튼은 행동을 명시하는 '탈퇴하기' 버튼으로 변경
