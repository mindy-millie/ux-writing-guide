# 세부 표기법

> **담당자**: @mindy (민디/서비스기획팀)
>
> 일관된 사용자 경험을 위해 텍스트에 사용하는 기호, 숫자, 띄어쓰기, 이모지의 표기 규칙을 정의합니다.

## 1. 기호

### 1-1. 문장 부호

| 기호 | 사용 규칙 | ✅ 좋은 예 | ❌ 나쁜 예 |
| --- | --- | --- | --- |
| 마침표 `.` | 문장 끝에 사용, UI 요소는 예외 | 결제가 완료되었습니다. | 로그인. |
| 쉼표 `,` | 나열, 문장 흐름이 필요할 때만 | 책, 오디오북을 볼 수 있어요 | 과도한 쉼표 사용 |
| 느낌표 `!` | 이벤트/프로모션에만, 1개까지 | 지금 시작해 보세요! | 지금 가입하세요!!! |
| 물음표 `?` | 실제 질문에만, 1개까지 | 삭제하시겠어요? | 삭제하시겠어요??? |
| 말줄임표 `...` | 텍스트 사용 ❌ (UI로만 표현) | - | 기다려 주세요… |
| 콜론 `:` | 라벨/값 구분, 시간 표기 | 결제일: 2026.05.06 | 결제일 : 2026.05.06 |
| 앰퍼샌드 `&` | 고유명사만 사용 | AT&T, 작가와 독자 | 책&오디오북 |
| 가운뎃점 `·` | 메타 정보 나열 | 작가 · 2024 · 300쪽 | 작가, 2024, 300쪽 |
| 슬래시 `/` | 한 묶음, 분수 | 도서명/저자 | 과도한 사용 |
| 세로막대 `\|` | UI에서만 사용 | (UI 구분자) | 텍스트에 사용 |
| 불릿 `•` | 텍스트에서는 • 사용 | • 항목1 | - 항목2 |

### 1-2. 따옴표

| 기호 | 사용 원칙 | 예시 |
| --- | --- | --- |
| 큰따옴표 `" "` | 문장을 인용할 때만 사용합니다. | 작가는 "다음 화도 기대해 주세요"라고 전했습니다. |
| 작은따옴표 `' '` | 특정 용어를 가리킬 때 사용합니다. | '회차순' 정렬을 선택하세요. |

### 1-3. 괄호

| 기호 | 용도 | 사용 규칙 | 예시 |
| --- | --- | --- | --- |
| `( )` 소괄호 | 보충 설명 | 중첩 사용 ❌ / 괄호 안 마침표 ❌ / 앞 단어와 붙여쓰기 / 문장+괄호+마침표 순서 | 무료체험(7일) |
| `《 》` 겹화살괄호 | 책 제목 | - | 《해리포터와 마법사의 돌》 |
| `[ ]` 대괄호 | 기능명, 이동 경로, 말머리 | - | [내서재] → [독서 통계] |
| `{ }` 중괄호 | 사용 금지 | - | - |
| `< >` 홑화살괄호 | 콘텐츠 제목, 책 제목 | 화살표 용도로 사용 금지 ❌ (화살표는 MDS 아이콘 사용) | ✅ \<오늘의 추천 작품\> |

### 1-4. 이동 경로 표기

**표기 방식**: [메뉴명 → 메뉴명 → 메뉴명]

> ✅ [마이페이지 → 작품 관리 → 에피소드 목록]

## 2. 숫자

### 2-1. 날짜

| 형식 | 표기 | 비고 |
| --- | --- | --- |
| 숫자 표기 | YYYY.MM.DD | 자릿수 유지 (2026.05.06) |
| 한글 표기 | YYYY년 MM월 DD일 | 0 미포함 권장 (2026년 5월 6일) |

**예외 표현**

| ❌ 사용 X | ✅ 권장 |
| --- | --- |
| 1주일, 2주일 | 일주일 |
| 1달 | 한달 |

### 2-2. 요일

| 형식 | 표기 예시 |
| --- | --- |
| 요약형 | 2026.05.06(수) |
| 풀어쓰기 | 2026년 5월 6일 수요일 |

### 2-3. 시간

| 형식 | 표기 | 비고 |
| --- | --- | --- |
| 숫자 표기 | HH:MM | 자릿수 유지 (09:05) |
| 한글 표기 | HH시 MM분 | 0 미포함 (9시 5분) |

> 💡 24시간제로 표기합니다.

### 2-4. 금액

천 단위 콤마(,)를 사용하고, 단위는 숫자 뒤에 붙여서 표기합니다.

> ✅ 1,500원 / 12,000코인
> ❌ 1500 원 / 12,000 코인

### 2-5. 범위

물결표 `~`를 사용하며 띄어쓰기 없이 표기합니다.

| 조건 | 표기 방식 | 예시 |
| --- | --- | --- |
| 단위가 같을 때 | 단위를 한 번만 표기 | 1~3분 |
| 단위가 다를 때 | 각각 표기 | 30분~1시간 |

### 2-6. 전화번호

| 형식 | 표기 |
| --- | --- |
| 국내 | 010-1234-5678 |
| 국제 코드 포함 | +82-10-1234-5678 |

> 💡 하이픈(-)으로 구분합니다.

### 2-7. 등록 시간 표기 (현재 기준)

서비스 내 "n분 전 / n일 전 등록" 등의 시간 표시는 아래 기준을 따릅니다.

| 경과 시간 | 표기 방식 | 예시 |
| --- | --- | --- |
| 1분 이내 | 방금 전 | 방금 전 |
| 59분 이내 | n분 전 | 12분 전 |
| 24시간 이내 | n시간 전 | 5시간 전 |
| 24시간 이후 | YYYY.MM.DD HH:MM | 2026.05.05 14:30 |

> 💡 남은 시간 표기: 'n일 남음' 형식 사용

### 2-8. 영문

**외래어** — 국립국어원 표기 기준을 따릅니다.

| 원칙 | 예시 |
| --- | --- |
| 불필요한 외래어는 우리말로 | 오픈 예정 → 공개 예정 |
| 굳어진 외래어는 한글로 표기 | 다운로드, 콘텐츠, 시스템, 인터페이스 |

**약어**

| 원칙 | 예시 |
| --- | --- |
| 기본은 대문자 표기 | API, URL, FAQ |
| 고유 표기는 대소문자 유지 | iOS, macOS, eBook |

**표기 예외**

> ✅ E-INK, EPUB

## 3. 띄어쓰기

| 항목 | 원칙 |
| --- | --- |
| 고유명사 | 한글 맞춤법을 따르지 않는 경우, 예외 케이스를 별도로 수집·관리합니다. |
| 본용언 + 보조용언 | 띄어쓰기 규칙을 학습하여 일관되게 적용합니다. |

> 💡 띄어쓰기 예외 케이스는 별도 시트에서 관리하며, 신규 케이스 발견 시 추가합니다.

## 4. 이모지

| 원칙 | 설명 | 예시 |
| --- | --- | --- |
| 개수 제한 | 한 문장 또는 한 영역에 1개만 사용합니다. | ✅ 새 에피소드가 등록됐어요 ✨ |
| 위치 | 텍스트 중간에 사용하지 않고, 맨 앞 또는 맨 뒤에만 배치합니다. | ✅ 🎉 오늘의 추천 / ❌ 오늘의 🎉 추천 |
| 간격 | 텍스트와 이모지 사이는 띄어 씁니다. | ✅ 환영합니다 👋 |
| 마침표 처리 | 문장 뒤에 이모지를 붙일 때는 마침표를 생략합니다. | ✅ 작품을 등록했어요 ✍️ / ❌ 작품을 등록했어요. ✍️ |
